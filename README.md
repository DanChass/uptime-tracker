🚀 Uptime Tracker
Сервис для высокопроизводительного мониторинга доступности веб-ресурсов, написанный на Go.

Проект разрабатывается как учебный пет-проект с упором на изучение конкурентности в Go (Goroutines), работу с сетью и построение надежной архитектуры бэкенд-приложения.

🛠 Основные возможности (Roadmap)
[x] Этап 1: Инициализация проекта и проектирование базовых моделей данных.

[ ] Этап 2: Синхронная проверка HTTP-статусов.

[ ] Этап 3: Асинхронный движок проверок (Worker Pool на горутинах).

[ ] Этап 4: Хранение истории проверок в базе данных (PostgreSQL/SQLite).

[ ] Этап 5: REST API для управления списком сайтов.

[ ] Этап 6: Система алертов через Telegram Bot.

[ ] Этап 7: Реализация Graceful Shutdown (безопасное завершение работы).

[ ] Этап 8: Контейнеризация проекта (Docker).

🏗 Структура проекта
Проект придерживается стандарта Standard Go Project Layout:

/cmd/tracker — точка входа в приложение (main.go).

/internal/models — описание бизнес-сущностей и моделей данных.

🚀 Как запустить (текущая версия)
Для запуска проекта в консоли выполните:

Bash
go run ./cmd/tracker/main.go
💻 Стек технологий
Language: Go (1.22+)

Concurrency: Goroutines, Channels, SyncGroup

Database: PostgreSQL (planned)

DevOps: Docker (planned)
